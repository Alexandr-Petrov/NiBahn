◆ Reelection_forced()

u8_t Reelection_forced ( u8_t  ownId,
short  reelection_delay,
u8_t  maxnibos 
)

This function forces reelection process intended to be used to return excluded from the chain Nibo back in the chain or to restore the structure of the chain if some Nibo is down.

Parameters
ownIdis the current ID of Nibo
reelection_delayis the predefined reelection delay for all Nibos
maxnibosis the max number of Nibos in the chain
Returns
0b01 to make Nibo go through the normal ID claim process

if ID of the Nibo is 0 (nibo was not taking part in the election

set ID as the max possible ID

disables uart0 port so Nibo would not recieve bytes from other Nibos before delay ends

time of the delay depends on the ID of the Nibo. Nibos with lower IDs will be able to participate sooner in this function all Nibos will return to their previous Roles if no changes were made to the chain before reelection

enables the uart0 port and now Nibo can recieve and send messages to other Nibos
